# Mango Live Sports Dashboard - API Documentation

Welcome to the API Documentation for the Mango Live Sports Dashboard. This document outlines the available REST API endpoints for fetching real-time match results, AI predictions, and exporting formatted prediction tips for Telegram channels or other webhook integrations.

---

## 🔮 1. Predictions Export API

### `GET /api/public/predictions/export`
Generates a formatted report of predictions for the latest round or daily tips. Supports both structured JSON output and pre-formatted, emoji-enriched markdown plain text ready for copy-pasting or posting to Telegram.

#### Request Parameters:

| Parameter | Type | Required | Default | Allowed Values | Description |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `type` | String | No | `live` | `live`, `daily` | Use `live` to get the latest database round predictions. Use `daily` to retrieve generated AI daily tips. |
| `league` | String | No | *None* | e.g. `England League` | Filters predictions by a specific league. Matches deterministically. |
| `format` | String | No | `text` | `text`, `json` | Use `text` to get pre-formatted Markdown plain text with emojis. Use `json` to get raw structured JSON. |

## 📋 2. Public Results Dashboard API

### `GET /api/public/results`
Returns paginated list of archived virtual football matches, grouped by date.

#### Request Parameters:

| Parameter | Type | Required | Default | Description |
| :--- | :--- | :--- | :--- | :--- |
| `page` | Integer | No | `1` | Pagination page number. |
| `pageSize` | Integer | No | `3` | Number of date blocks to fetch per page. |
| `league` | String | No | *None* | Filter results by league name. |
| `dateFrom` | String | No | *None* | Start date filter in DD/MM/YYYY format. |
| `dateTo` | String | No | *None* | End date filter in DD/MM/YYYY format. |

#### Response Body Example (JSON):

## 📡 3. Live Odd Streams & Realtime Updates

### `GET /api/live-stream`
Server-Sent Events (SSE) endpoint that streams scraper updates in real time to the browser client. The client maintains an active persistent connection to receive instant updates.

#### Push Payload Formats:
- **`initializing` Status**: Scrapers are initializing or navigating.
- **`live` Status**: Scrapers pushed updated scoreboard data.

---

## ⚙️ 4. Admin Management Endpoints

### `POST /api/scraper/reload`
Signals the backend results and odds scrapers to refresh their connections and configurations.

### `GET /api/admin/vfootball/sync-all`
Forces a high-speed native sync on demand. Triggers browser scraper tasks to immediately scrape Virtual Football leagues.
